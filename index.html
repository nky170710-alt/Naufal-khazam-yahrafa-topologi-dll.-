Pengenalan Topologi Jaringan

Topologi jaringan adalah susunan fisik atau logis perangkat dalam suatu jaringan. Mempelajari topologi membantu perancang jaringan menentukan strategi koneksi, redundansi, keamanan, dan estimasi biaya.

Pada materi ini: pembahasan mendalam berbagai topologi (Bus, Star, Ring, Mesh, Tree, Hybrid, Peer-to-Peer, Point-to-Point, Point-to-Multipoint, Daisy Chain, Cellular, Wireless, Extended Star, Partial Mesh, Hierarchical, VLAN & Virtual Topologies, dan lain-lain).

Tujuan Pembelajaran

  • Memahami karakteristik tiap topologi
  • Membandingkan kelebihan & kekurangan
  • Menentukan topologi yang sesuai kebutuhan
  • Mengenal teknologi kabel (UTP, Coaxial, Fiber) & nirkabel

Topologi Bus

Topologi Bus memakai satu kabel utama (backbone). Semua perangkat terhubung ke kabel tersebut; sinyal yang dilewatkan dapat diterima oleh semua node, namun hanya node tujuan yang memprosesnya.

Rincian & praktik

  • Umumnya menggunakan terminator di kedua ujung kabel untuk mencegah pantulan.
  • Cocok untuk jaringan kecil/eksperimental; tidak ideal untuk jaringan modern dengan traffic tinggi.

Kapan dipakai?

Paling sesuai untuk lab kecil atau setup sementara.

Topologi Star

Setiap node terhubung ke perangkat pusat (hub/switch). Switch managed memberikan kontrol VLAN dan QoS, sedangkan hub sederhana mengirim broadcast ke semua port.

Rekomendasi Implementasi

Gunakan switch berkapasitas port lebih besar dan uplink gigabit untuk skalabilitas. Redundansi: DOUBLe uplink ke core switch untuk fault tolerance.

Topologi Ring

Node tersusun melingkar; data bergeser dari satu node ke node berikutnya. Token passing adalah mekanisme untuk mengendalikan akses transmisi sehingga collision bisa diminimalkan.

Keandalan

Versi dual-ring menyediakan jalur backup jika satu sisi putus.

Topologi Mesh (Full & Partial)

Mesh penuh: setiap node terhubung langsung ke semua node lain ‚Äî memberikan redundansi maksimal. Partial mesh: hanya node utama (gateway/core) yang saling terhubung untuk menekan biaya.

Penggunaan

Sering dipakai pada backbone ISP, pusat data, infrastruktur kritikal.

Topologi Tree

Hierarki gabungan star dan bus‚Äîcocok untuk organisasi besar dengan cabang & sub-departemen.

Tips Desain

  • Rencanakan kapasitas uplink antar tingkat; gunakan link aggregate/port-channel jika diperlukan.

Topologi Hybrid

Kombinasi topologi sesuai kebutuhan. Misal: star untuk LAN, mesh untuk backbone antar gedung.

Pertimbangan

Dokumentasi & monitoring lebih penting karena kompleksitas tinggi.

Peer-to-Peer (P2P)

Desentralisasi‚Äîsetiap node juga bertindak sebagai server. Cocok untuk kolaborasi kecil, tetapi kontrol & keamanan lebih menantang.

Point-to-Point & Point-to-Multipoint

PtP: link langsung antar dua node (leased line, fiber link). PtMP: satu node utama melayani banyak node (mis. wireless AP ke klien).

Contoh

PtP untuk koneksi kantor-cabang; PtMP untuk distribusi wifi publik atau microwave link ISP.

Daisy Chain

Node tersusun berantai. Simpel tapi rentan; outage di tengah rantai memutus koneksi downstream.

Topologi Cellular

Digunakan operator seluler: area dibagi sel, tiap sel dilayani base station. Handover & frequency reuse penting dalam desain.

Wireless (WLAN & Wifi Mesh)

WLAN biasanya infrastruktur (AP + controller) atau mesh (AP saling terhubung). Site survey & channel plan sangat penting.

Hierarchical / 3-tier

Access ‚Üí Distribution ‚Üí Core. Memudahkan policy, QoS, dan segmentasi jaringan.

Extended Star

Multiple star clusters dihubungkan melalui backbone. Cocok untuk gedung bertingkat / kampus.

Partial Mesh

Hubungkan node kritikal (core, gateway, data center) secara mesh; edge nodes ke core. Efektif biaya & andal.

Virtual Topologies (VLAN / SDN)

Topologi logis: VLAN memisahkan domain broadcast, SDN memisahkan control plane dan data plane‚Äîmemungkinkan topologi virtual yang fleksibel.

Keuntungan

  • Fleksibilitas, segmentasi keamanan, manajemen terpusat.

Deployment & Best Practices

Rencanakan kabel, power, rack, cooling, monitoring (SNMP/Telemetry), dan backup link. Dokumen topologi & IP plan (CIDR/VLSM) wajib ada.

Contoh Implementasi & Studi Kasus

Studi: kantor 200 user, pilih extended star + partial mesh backbone; kampus: tree dengan backbone fiber; ISP: core mesh penuh antar POP.

Advanced: Redundancy & Routing

Pemilihan routing (OSPF/BGP/MPLS) dan protokol agregasi (LACP), peran NAT/Firewall & DMZ, serta strategi failover (VRRP, HSRP).

Tools & Resources

Rekomendasi: Wireshark, Packet Tracer, GNS3, NetBox untuk dokumentasi, serta site survey tools untuk wifi.

Desain Pembelajaran ‚Äî Jaringan & Topologi

Capaian Pembelajaran (CP): Menerapkan perencanaan topologi, arsitektur jaringan sesuai kebutuhan serta pengalamatan jaringan (subnetting, CIDR, VLSM).

Kompetensi: Siswa mampu merancang topologi, melakukan perhitungan subnetting (CIDR & VLSM), memilih media kabel (UTP, Coaxial, Fiber) dan teknologi nirkabel serta menguji instalasi dasar.

Materi Utama

  • Konsep topologi & pemilihan topologi
  • Subnetting: binary, CIDR, VLSM
  • Teknologi kabel: UTP (Cat5/5e/6), Coaxial, Fiber Optic
  • Teknologi nirkabel: Wi-Fi modes, site survey, channel planning
  • Perancangan IP plan & addressing
  • Keamanan dasar & manajemen jaringan

Subnetting singkat (CIDR & VLSM)

CIDR (Classless Inter-Domain Routing) memungkinkan pemakaian notasi prefix (mis. /24) untuk fleksibilitas alokasi IP. VLSM (Variable Length Subnet Mask) memungkinkan subdivisi jaringan menjadi subnetwork dengan ukuran berbeda sesuai kebutuhan.

Contoh ringkas: Dari 192.168.0.0/24 dapat dibuat subnet /26 untuk subnetwork kecil (masing-masing menampung 62 host).

Teknologi Kabel

  • UTP (Twisted Pair): Cat5e/Cat6 untuk LAN. Mudah instalasi, ekonomis.
  • Coaxial: Dipakai di beberapa legacy network / CCTV.
  • Fiber Optic: Untuk backbone & jarak jauh; sangat tinggi bandwidth & tahan gangguan elektromagnetik.

Rencana Pembelajaran Saran

  • Teori: pengenalan topologi & IP addressing
  • Praktikum: instalasi kabel UTP, konfigurasi switch dasar
  • Latihan: perhitungan subnetting CIDR & VLSM
  • Proyek akhir: desain topologi & implementasi mini-LAN

Kesimpulan

Pemilihan topologi harus mempertimbangkan kebutuhan spesifik: ukuran jaringan, kebutuhan redundansi, anggaran, dan kemudahan pengelolaan. Untuk infrastruktur mission-critical, pilih arsitektur dengan redundancy (mesh/partial mesh/hybrid). Untuk skala SME, star/extended star biasanya cukup.
